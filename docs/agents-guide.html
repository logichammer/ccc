🤖 Agent System Guide

Claude Code Templates includes 20+ specialized agents that work together to provide comprehensive development, analysis, and optimization capabilities. This guide covers how agents work, when they're triggered, and how to leverage the orchestration system.

🎯 Agent Orchestration Overview

🏗️ SystemArchitectAgent (Default Orchestrator)

Orchestration Pattern: Lead-SubAgent with Fan-out/Fan-in
Primary Objective: Plan → Delegate → Integrate

Typical Flow:
1. 🏗️ Analyzes requirements and plans approach
2. 🔄 Hands off to specialized agents for execution
3. 🏗️ Integrates results and coordinates next steps
4. 🔄 Continues until task completion
Agent Handoff Example:

User Request: "Build user authentication with tests"
↓
🏗️ [SystemArchitectAgent] → Plan overall approach
↓
🔄 HANDOFF: SystemArchitectAgent → FeatureDeveloperAgent
↓
⚡ [FeatureDeveloperAgent] → Implement authentication code
↓
🔄 HANDOFF: FeatureDeveloperAgent → QualityTesterAgent
↓
🧪 [QualityTesterAgent] → Generate test suite
↓
🔄 HANDOFF: QualityTesterAgent → SystemArchitectAgent
↓
🏗️ [SystemArchitectAgent] → Final integration and summary

🎮 Advanced Agent Usage

🔄 Parallel Agent Execution

For complex tasks, multiple agents can work in parallel:

Example: "Optimize this application for performance and security"

🏗️ SystemArchitectAgent plans the approach
↓
⚡ PerformanceProfilerAgent ║ 🔒 SecurityAuditorAgent
(Running in parallel)
↓
🏗️ SystemArchitectAgent integrates findings
↓
⚡ FeatureDeveloperAgent implements optimizations
↓
✅ DeliveryVerifierAgent validates improvements
💡 Pro Tips for Agent Orchestration:

• Trust the System: Let SystemArchitectAgent handle orchestration automatically
• Be Specific: Use trigger keywords to influence agent selection
• Context Matters: Agents consider project type and current context
• Quality Gates: Agents enforce quality checks before handoffs
• Rollback Capability: Agents can revert changes if issues arise
