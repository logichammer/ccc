üéØ Workflow Best Practices

General Workflow Principles

  • Agent Orchestration: Let Claude automatically choose and switch between appropriate agents
  • Progressive Enhancement: Start with basic implementation and enhance iteratively
  • Quality Gates: Each workflow includes built-in quality checks and validation
  • Documentation First: All workflows generate documentation alongside implementation
  • Reversible Changes: External project workflows are completely reversible
  • Data Preservation: SEO and analysis data is properly organized and preserved

Troubleshooting Workflows

When workflows encounter issues:

  • Check Dependencies: Ensure required tools and MCPs are available
  • Verify Permissions: Check file and directory permissions for project setup
  • Review Configuration: Validate config.toml settings and paths
  • Agent Handoffs: Monitor agent transitions for stuck or failed handoffs
  • Fallback Options: Most workflows have alternative approaches when primary methods fail
