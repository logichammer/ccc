CCC Workflow Format Specification

This document defines the markdown format for creating initial workflow drafts that will be processed by the CCC Workflow System.

Workflow Creation Process

  1. Write Initial Draft: Create a markdown file describing your workflow goals in natural language
  2. Chat Interview: System will ask clarifying questions via chat interface
  3. System Generates Suggestions: Detailed workflow with agents, MCPs, and execution plan
  4. User Edits & Approves: Modify the suggested workflow as needed
  5. Technical Validation: System verifies feasibility and dependencies
  6. Execute Workflow: Run the approved workflow

Initial Draft Format

Basic Structure

# [Workflow Name]

## Goal
Brief description of what you want to accomplish.

## Context
- What type of project/client is this for?
- What data/files do you have available?
- What kind of output do you need?

## Requirements (Optional)
- Specific constraints or preferences
- Output format preferences
- Time sensitivity

## Expected Challenges (Optional)
- Known limitations or concerns
- Data quality issues
- Integration requirements

Example Initial Drafts

SEO Audit Example

# SEO Technical Audit for Client

## Goal
I need to perform a comprehensive technical SEO audit for a client's e-commerce website.

## Context
- E-commerce site built on Shopify
- I have access to Google Search Console data
- Client wants to improve Core Web Vitals and organic rankings
- Need professional report for client presentation

## Requirements
- Must include competitor analysis (I'll provide competitor list)
- Need both technical recommendations and content gaps
- Output should be professional HTML report + Excel data
- Timeline: Complete within 2 hours

## Expected Challenges
- Site has 2000+ products, may be slow to crawl
- GSC data might be incomplete for newer pages
- Need to balance depth vs speed of analysis

Python Code Analysis Example

# Code Quality Assessment for Flask API

## Goal
Analyze a Python Flask API codebase for security, performance, and maintainability issues.

## Context
- Flask application with ~5000 lines of code
- Uses SQLAlchemy, Redis, Celery
- Production deployment needs security review
- Team wants test coverage improvements

## Requirements
- Focus on security vulnerabilities first
- Need specific test case recommendations
- Want performance bottleneck identification
- Must include dependency audit

## Expected Challenges
- Some legacy code without type hints
- Mixed coding standards across modules
- Large SQLAlchemy models may be complex to analyze

Data Correlation Analysis Example

# Customer Behavior Correlation Analysis

## Goal
Find unexpected correlations in customer data to identify conversion optimization opportunities.

## Context
- E-commerce customer data (purchase history, demographics, behavior)
- 6 months of data, ~50k customers
- Want to improve conversion rates and AOV
- Multiple CSV files with different data points

## Requirements
- Statistical significance testing
- Visual correlation matrices
- Actionable insights prioritized by potential impact
- Export findings to presentation-ready format

## Expected Challenges
- Data quality may vary between sources
- Need to handle missing values appropriately
- Some seasonal effects may confuse correlations

What Happens Next

After you create your initial draft markdown file:

  1. System Analysis: The workflow system will analyze your draft and identify:
    • Available agents that match your requirements
    • Relevant MCPs for data collection
    • Input file expectations
    • Potential workflow stages
  2. Interview Questions: Based on the analysis, you'll be asked clarifying questions such as:
    • "Which specific competitor URLs should be analyzed?"
    • "What's the acceptable processing time for this workflow?"
    • "Do you need real-time data or is cached data acceptable?"
    • "Which output format is most important: HTML report, Excel data, or JSON?"
  3. Generated Workflow: A detailed workflow specification including:
    • Specific agent assignments per stage
    • MCP selections with alternatives
    • Parallelization opportunities and warnings
    • Input file requirements
    • Expected output structure
    • Estimated execution time
Important Notes:
• Keep initial drafts conversational and focused on business goals
• Don't worry about technical implementation details in the draft
• The more context you provide, the better the system can tailor recommendations
• You'll have full control to edit the generated workflow before execution

File Storage Locations

  • Initial Drafts: /workflows/draft/[workflow-name].md
  • Generated Suggestions: /workflows/suggested/[workflow-name].md
  • Approved Workflows: /workflows/active/[workflow-name].md
  • Completed Workflows: /workflows/archive/[workflow-name].md (optional)
