üåü System Overview

The CCC Workflow System is a sophisticated markdown-based workflow definition and execution engine that enables you to:

  • Define Complex Workflows - Write workflows in markdown with variable substitution
  • Multiple Format Styles - From conversational to enterprise-grade technical specifications
  • JSON Configuration - Separate configuration from workflow logic
  • Auto-Detection - Smart single-file processing with helpful error handling
  • Interview-Driven Creation - Interactive refinement of workflows
  • Mock Execution - Professional simulation with realistic outputs
  • Agent Integration - Automatic agent and MCP recommendations
üéØ Core Philosophy: One workflow draft at a time for clean development, with powerful auto-detection and zero-argument commands for simplicity.

üöÄ Quick Start Guide

Basic Workflow Process

# 1. Create a draft workflow (markdown file)
echo "# My SEO Audit" > workflows/draft/my-seo-audit.md

# 2. Generate JSON config template (auto-detects single file)
workflow-system config-template

# 3. Edit the generated config with your values
nano workflows/draft/inputs/workflow-config.json

# 4. Create workflow through interactive interview
workflow-system create

# 5. Execute the workflow
workflow-system execute my-seo-audit
‚ú® That's it! The system handles auto-detection, configuration loading, agent recommendations, and execution simulation automatically.

üéÆ CLI Commands Reference

Core Commands

workflow-system create

Creates a workflow from draft files through interactive interview.

workflow-system create # Auto-detect single draft
workflow-system create my-workflow # Explicit workflow name
workflow-system create --client acme # Client-specific settings

workflow-system config-template

Generates JSON configuration templates for workflows.

workflow-system config-template # Auto-detect single draft
workflow-system config-template my-draft # Explicit workflow name

workflow-system validate

Validates technical feasibility and requirements.

workflow-system validate my-workflow # Check if workflow is executable
workflow-system validate --verbose # Detailed validation output

workflow-system execute

Executes workflows with professional simulation.

workflow-system execute my-workflow # Full execution
workflow-system execute my-workflow --dry-run # Preview only

workflow-system list

Lists all workflows across all stages.

workflow-system list # Show all workflows with status

workflow-system archive

Archives completed workflows.

workflow-system archive my-workflow # Move to archive

workflow-system status

Shows system status and configuration.

workflow-system status # System overview

Auto-Detection Rules

üìã Single File Rule: Commands like create and config-template auto-detect when exactly one draft file exists.
  • ‚úÖ One Draft File: Auto-detects and processes
  • ‚ùå Multiple Draft Files: Lists options and requires explicit name
  • ‚ùå No Draft Files: Helpful error with guidance

üìù Workflow Format Variations

The system supports multiple workflow format styles, from simple conversational to enterprise-grade technical specifications:

Format Categories

üó£Ô∏è Conversational Formats

  • Minimal Request - Ultra-simple, just provide URL and keyword
  • Conversational - Natural language descriptions
  • Narrative Story - Engaging detective-style presentation

üîß Technical Formats

  • Technical Brief - YAML configs with tool specifications
  • Detailed Spec - JSON schemas and comprehensive parameters
  • Code Style - Programming syntax with function definitions

üè¢ Enterprise Formats

  • Enterprise Spec - Full compliance, SLAs, quality assurance
  • Scientific Method - Research protocols with statistical validation
üí° Key Insight: All formats deliver the same high-quality results - they differ only in presentation complexity and initial request style.

Variable System

All formats support variable substitution using {{VARIABLE_NAME}} syntax:

  • {{SITE_URL}} - Target website
  • {{MAIN_KEYWORD}} - Primary keyword
  • {{ANALYSIS_DEPTH}} - Complexity level
  • {{COMPETITOR_COUNT}} - Number of competitors

üîß Troubleshooting

Common Issues

‚ùå Multiple draft files found
Solution: Either specify explicit workflow name or keep only one draft file at a time for auto-detection.
‚ùå No draft workflow files found
Solution: Create a markdown workflow file in workflows/draft/ directory.
‚ùå Config template variables not detected
Solution: Ensure variables use {{VARIABLE_NAME}} format in markdown.

Debug Commands

workflow-system status --verbose # Detailed system status
workflow-system validate workflow --verbose # Detailed validation
workflow-system execute workflow --dry-run # Preview execution
