üåü Overview

The CCC Workflow System uses an intelligent interview process to transform basic workflow drafts into comprehensive, executable specifications. This chat-based system gathers requirements, recommends agents, and generates detailed workflow implementations.

üéØ Core Principle: The interview system bridges the gap between simple draft workflows and complex execution requirements through intelligent question-driven refinement.

üîÑ Interview Flow Process

Draft Analysis

  • Parses markdown workflow file
  • Loads JSON configuration (if present)
  • Extracts variables and requirements
  • Identifies workflow complexity level

Project Context Gathering

  • Project type identification
  • Timeline and urgency assessment
  • Resource availability evaluation
  • Quality vs. speed trade-offs

Requirements Deep Dive

  • Input file analysis and validation
  • Data source requirements
  • Special constraints or preferences
  • Output format specifications

Technical Assessment

  • Agent capability analysis
  • MCP server availability check
  • Tool integration requirements
  • System resource evaluation

Agent Recommendations

  • Primary agent selection
  • Supporting agent identification
  • MCP integration planning
  • Workflow orchestration design

Workflow Generation

  • Structured workflow creation
  • Stage dependency mapping
  • Success criteria definition
  • Quality validation rules
