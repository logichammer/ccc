🎯 5-Minute Workflow

Create Your First Workflow

Start with a simple SEO audit workflow. Copy and paste this into your terminal:

cat > workflows/draft/my-first-seo-audit.md << 'EOF'
# My First SEO Audit

## Goal
Complete SEO analysis for {{SITE_URL}} targeting "{{MAIN_KEYWORD}}"

## Process
- Crawl site and analyze technical structure
- Research top competitors for target keyword
- Run performance tests on key pages
- Generate comprehensive report with recommendations

## Expected Output
- Interactive dashboard comparing site vs competitors
- Prioritized action plan with implementation guidance
- Technical report with specific fixes and improvements
EOF

✅ Result: You now have a draft workflow ready for configuration!

Generate Configuration Template

The system will automatically detect your draft file and create a configuration template:

workflow-system config-template

Expected Output:

🎯 Auto-detected draft workflow: my-first-seo-audit
✅ Config template generated!
📁 Location: /workflows/draft/inputs/workflow-config.json
📝 Edit this file to provide actual values for:
• SITE_URL: https://example.com
• MAIN_KEYWORD: target keyword phrase
Configure Your Variables

Edit the generated configuration with your actual values:

nano workflows/draft/inputs/workflow-config.json

Replace the example values:

{
"workflow_name": "my-first-seo-audit",
"variables": {
"SITE_URL": "https://your-website.com",
"MAIN_KEYWORD": "your target keyword"
},
"client_settings": {
"client_slug": "my-client",
"output_directory": "/home/loki/seo-projects/my-client/",
"archive_previous": true
}
}

💡 Tip: Use Ctrl+X, then Y, then Enter to save in nano

Create Your Workflow

Run the interactive interview to create your complete workflow:

workflow-system create

Sample Interview Responses:

🎯 Auto-detected draft workflow: my-first-seo-audit

📋 PROJECT CONTEXT
Select timeline (1-3) [default: 2]: 2 # Standard timeline
Select resources (1-3) [default: 2]: 2 # Standard resources

📁 INPUT REQUIREMENTS
Do you have any input files available? [y/N]: n # No additional files

🤖 AGENT & TOOL RECOMMENDATIONS
Proceed with these recommendations? [Y/n]: y # Accept recommendations

🎨 Try Different Workflow Formats

The system supports multiple workflow format styles. Try these examples:

💬 Conversational Format

Natural language, friendly interaction

cat > workflows/draft/conversational-audit.md << 'EOF'
# Friendly SEO Check

Just tell me your website {{SITE_URL}} and keyword "{{MAIN_KEYWORD}}" and I'll analyze everything for you!

What you'll get:
- Complete performance overview
- Easy-to-understand recommendations
- Clear next steps
EOF

🔧 Technical Format

Detailed specifications, tool callouts

cat > workflows/draft/technical-audit.md << 'EOF'
# Technical SEO Analysis

## Configuration
```yaml
target_site: "{{SITE_URL}}"
primary_keyword: "{{MAIN_KEYWORD}}"
crawl_budget: 500
analysis_mode: "comprehensive"
```

## Tools
- Firecrawl for site crawling
- DataForSEO for SERP analysis
- Lighthouse for performance testing
EOF

📋 Action-Oriented Format

Bullet points, clear structure

cat > workflows/draft/action-audit.md << 'EOF'
# SEO Action Plan

## What You Need
• Website URL: {{SITE_URL}}
• Target keyword: {{MAIN_KEYWORD}}

## What Happens
• Site crawl and analysis
• Competitor benchmarking
• Performance testing
• Action plan creation
EOF

🔄 Multiple Workflows

Manage different workflow types:

# Create different workflows for different purposes
cp workflows/draft/my-first-seo-audit.md workflows/draft/competitor-research.md
cp workflows/draft/my-first-seo-audit.md workflows/draft/performance-audit.md

# Generate configs for each
workflow-system config-template competitor-research
workflow-system config-template performance-audit

📊 Client Management

Organize work by client:

# Different client configurations
echo '{
"variables": {"SITE_URL": "https://client-a.com", "MAIN_KEYWORD": "their keyword"},
"client_settings": {"client_slug": "client-a"}
}' > workflows/draft/inputs/workflow-config-client-a.json

# Execute with specific client settings
workflow-system create --client client-a
workflow-system execute my-workflow --client client-a

🔍 Useful Commands

# See all your workflows
workflow-system list

# Check system status
workflow-system status

# Validate before executing
workflow-system validate my-workflow

# Preview execution without running
workflow-system execute my-workflow --dry-run

# Run with detailed logging
workflow-system execute my-workflow --verbose

# Archive completed work
workflow-system archive my-workflow

🎯 Tips for Success

💡 Best Practices:
  • Start Simple: Begin with basic workflows, add complexity gradually
  • One Draft Rule: Keep only one draft file at a time for auto-detection
  • Always Configure: Use config-template to ensure all variables are set
  • Validate First: Run validate before executing to catch issues early
  • Use Dry Run: Preview execution with --dry-run for complex workflows
🚨 Common Gotchas:
  • Multiple Draft Files: Auto-detection requires exactly one draft file
  • Config Location: JSON config must be in inputs/ directory
  • Variable Format: Use {{VARIABLE}} with double braces
  • Interview Mode: Run in interactive terminal, not scripts
