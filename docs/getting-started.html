üìã Understanding the CCC Commands

Important: CCC commands set up your project for Claude development - they don't start Claude automatically. You still need to open Claude Code and navigate to your project directory.

üîß ccc-work Command (claude-work)

üìÅ

What it DOES:

  • ‚úÖ Auto-detects project type (Python, WordPress, SEO)
  • ‚úÖ Creates .claude/ directory with full toolkit
  • ‚úÖ Copies all 23 agents to your project
  • ‚úÖ Adds appropriate CLAUDE.md (Python/SEO/WordPress focused)
  • ‚úÖ Updates .gitignore to exclude .claude/ directory
  • ‚úÖ Works from current directory if you use ccc-work .
‚ùå

What it DOESN'T do:

  • ‚ùå Does NOT start Claude - you still need to open Claude Code
  • ‚ùå Does NOT turn on bypass permissions - that's a Claude Code setting
  • ‚ùå Does NOT remember project type - it auto-detects each time
  • ‚ùå Does NOT automatically navigate - you still cd to your project

Usage Examples:

# Super simple - just run in any project directory:
cd /mnt/c/python/dvl/Linky
ccc-work # Auto-detects type, uses current dir

# Force specific type on current directory:
ccc-work --type python # Forces Python toolkit on current dir

# Work with any directory anywhere:
ccc-work /path/to/any/project # Works with projects anywhere

# Get help and examples:
ccc-work --help # Full help with examples
ccc-work --list-examples # Detailed usage examples
ccc-help # Alias for examples

# Quick cleanup:
ccc-work --cleanup # Remove .claude/ from current dir
ccc-clean # Alias for force cleanup

üîß Troubleshooting Common Issues

Permission Denied

Problem: permission denied: claude-init
Solution: Make the script executable:
chmod +x /home/loki/code/ccc/bin/claude-init

Module Not Found: toml

Problem: ModuleNotFoundError: No module named 'toml'
Solution: Install the dependency:
pip install toml

Directory Not Found

Problem: Project directories don't exist
Solution: The system creates them automatically, but you can create manually:
mkdir -p /home/loki/{seo-projects,python-projects}
